📦 Basic Docker Commands
▼
🛠️ Image Management
# List all images docker images # Remove an image docker rmi <image-name> # Remove unused images docker image prune # Pull an image from Docker Hub docker pull openjdk:21 # Build an image from Dockerfile docker build -t <image-name> .
🏃 Container Operations
# Run a container docker run -p 8080:8080 <image-name> # Run container in detached mode (background) docker run -d -p 8080:8080 <image-name> # List running containers docker ps # List all containers (including stopped) docker ps -a # Stop a container docker stop <container-id> # Remove a container docker rm <container-id>
🔍 Debugging & Logs
# View container logs docker logs <container-id> # Follow logs in real-time docker logs -f <container-id> # Execute command inside running container docker exec -it <container-id> bash # Inspect container details docker inspect <container-id>
🧹 Cleanup Commands
# Remove all unused containers, networks, images docker system prune # Remove everything including volumes docker system prune -a --volumes # Remove all containers (force) docker rm -f $(docker ps -aq) # Remove all images docker rmi $(docker images -q)
🐳 Dockerfile for Spring Boot
▼
📄 Basic Dockerfile
# Use OpenJDK 21 as base image FROM openjdk:21-jdk-slim # Set working directory inside container WORKDIR /app # Copy the JAR file from target folder to container COPY target/fitness-app.jar myapp.jar # Expose the port your Spring Boot app runs on EXPOSE 8080 # Define the command to run your application ENTRYPOINT ["java", "-jar", "myapp.jar"]
🚀 Production-Ready Dockerfile
FROM openjdk:21-jdk-slim # Install curl for health checks RUN apt-get update && apt-get install -y curl && rm -rf /var/lib/apt/lists/* # Create non-root user RUN groupadd -r appuser && useradd -r -g appuser appuser WORKDIR /app # Copy JAR file COPY target/fitness-app.jar myapp.jar # Change ownership to non-root user RUN chown appuser:appuser myapp.jar # Switch to non-root user USER appuser # Health check HEALTHCHECK --interval=30s --timeout=3s --start-period=30s --retries=3 \ CMD curl -f http://localhost:8080/actuator/health || exit 1 EXPOSE 8080 # JVM optimization flags ENTRYPOINT ["java", "-Xms512m", "-Xmx1g", "-XX:+UseG1GC", "-jar", "myapp.jar"]
🚀 Step-by-Step Deployment
▼
Configure Maven Build

Add finalName to your pom.xml in the build section:

⚙️ pom.xml
<build> <finalName>fitness-app</finalName> <plugins> <plugin> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-maven-plugin</artifactId> </plugin> </plugins> </build>
Why? This ensures your JAR file has a consistent name (fitness-app.jar) instead of including version numbers.
Create Dockerfile

Create a file named Dockerfile in your project root (same level as pom.xml):

🐳 Dockerfile
FROM openjdk:21-jdk-slim WORKDIR /app COPY target/fitness-app.jar myapp.jar EXPOSE 8080 ENTRYPOINT ["java", "-jar", "myapp.jar"]
Build JAR File
💻 Command Line
# Clean and package mvn clean package

🎯 IntelliJ IDEA

  • Look for Maven tab on the right sidebar
  • Expand your project name
  • Expand Lifecycle
  • Double-click clean
  • Double-click package
Build Docker Image
🔨 Build Command
# Build the Docker image docker build -t fitness-app .
Make sure you run this command in the same directory where your Dockerfile exists!
Run Docker Container
▶️ Run Commands
# Run the container docker run -p 8080:8080 fitness-app # Or run in background (detached mode) docker run -d -p 8080:8080 fitness-app

Access your app: http://localhost:8080

🔧 Troubleshooting & Best Practices
▼
Common Issue: "java.net.ConnectException: Connection refused"
Solution: Use service names instead of localhost in Docker containers.
🔍 Debug Commands
# Check container logs docker logs <container-id> # View running processes in container docker exec <container-id> ps aux # Check network connectivity docker exec <container-id> ping eureka-server # View environment variables docker exec <container-id> env
  • Use slim images: Reduce attack surface and image size
  • Run as non-root: Never run containers as root user
  • Add health checks: Monitor container health automatically
  • Use .dockerignore: Exclude unnecessary files from build context
  • Optimize layers: Combine RUN commands to reduce image layers
